01 — Overview

What This System Does

Core Idea

All Playwright E2E tests live in a dedicated Skills Repo, completely separate from application code. Any application repo can fire a GitHub Actions workflow that checks out the Skills Repo, runs the relevant Playwright suites against the live dev server (no local stack needed), and emails a pass/fail report to the right people — all on demand, with zero setup per developer.

🏗️

What exists today

Application repos (API, Frontend, Admin API), GitHub Actions for deployment, seed data, and basic API test scripts.

🆕

What we're adding

A central Skills Repo with Playwright suites, a reusable run-e2e GitHub Actions workflow, and an email report step.

🌐

Test target

Tests always run against the live dev environment — dev.trustmyreferrals.com. No Docker, no local stack in CI.

📬

Reporting

On completion, an HTML report is uploaded as a workflow artifact and a pass/fail summary email is sent to configured recipients.

04 — System Components

What Each Component Does

🎯

Skills Repo New

Standalone GitHub repo containing all Playwright test suites, helpers, fixtures, and the reusable run-e2e.yml workflow. No app code lives here — only test logic and CI tooling.

📦

App Repos Exists

trustmyreferrals-api, trustmyreferrals-front, trust-admin-api. Each adds a thin trigger-e2e.yml workflow that calls the reusable workflow in the Skills Repo.

⚡

GitHub Actions Extends

Reusable workflow in Skills Repo accepts base_url, suite, and recipients as inputs. App repos call it as uses: indoobit-skills-repo/.github/workflows/run-e2e.yml.

🌐

Live Dev Environment Exists

dev.trustmyreferrals.com (Frontend) + api-dev.trustmyreferrals.com (API). Playwright hits HTTPS directly. The seeded test database provides predictable data for all personas.

📬

Email Service New

A Node.js report step in the workflow uses SendGrid or AWS SES to send a formatted HTML summary email. Pass/fail counts, anomaly flags, and artifact link are included.

📊

Anomaly Detection New

A lightweight step that compares this run's JSON results to the prior run saved as a workflow artifact. New failures, newly passing tests, and performance regressions are flagged in the email.

06 — Report & Notification

What the Report Contains

Email Anatomy

Every run produces one email. The subject line immediately signals status: ✅ E2E PASSED — Auth · Deck · Requirements (17/17 tests) or ❌ E2E FAILED — deck.spec: 2 failures (15/17 tests). The body includes suite-by-suite breakdown, anomaly section if regressions detected, and a direct link to the full HTML artifact in GitHub Actions.

✅

Pass Summary

Suite name · test count · duration · target URL · run timestamp. Sent to all recipients on every green run.

❌

Failure Detail

Which test failed · which assertion · screenshot filename attached · trace link. Sent with higher urgency styling.

🔍

Anomaly Flags

Tests that passed last run but failed this run are highlighted as regressions. Tests newly passing after repeated failure are noted as recoveries.

🔗

Artifact Link

Direct deep-link to the HTML Playwright report stored as a GitHub Actions artifact — retained 30 days, downloadable by anyone with repo access.

Recipient Configuration

Recipients are set per-environment via the EMAIL_RECIPIENTS GitHub secret (comma-separated). The reusable workflow accepts an override via the recipients input, so callers can target a specific team without touching the Skills Repo.

  • 1
    Run completes — exit code captured Playwright exits 0 (all pass) or 1 (any failure). Exit code drives subject line and email urgency.
  • 2
    results.json parsed by send-report.mjs Playwright's built-in JSON reporter writes test-by-test outcomes. The script reads this file, never screen-scrapes the HTML.
  • 3
    Anomaly diff runs (if prior artifact exists) diff-results.mjs downloads the previous run's results.json via the GitHub API and computes a delta.
  • 4
    HTML email assembled and sent Inline styles, clear pass/fail colour coding, suite table, anomaly section, and artifact link. Sent via SendGrid or AWS SES depending on what the project already uses.
  • 5
    HTML report uploaded as GitHub artifact Full interactive Playwright HTML report retained 30 days. Traces (only on failure) retained 7 days.
